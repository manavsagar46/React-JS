NPM (Node Package Manager)
- Manages node packages (but npm ≠ "node package manager").
- Packages are hosted on npm registry.
- Install using CLI.

Important Commands:
- npm init → creates package.json (project info + dependencies).
- npm install <pkg> → installs package.
- npm install -D <pkg> → install package as dev dependency.

Extra Files:
- node_modules/ → all installed packages.
- package-lock.json → exact versions (ensures consistent installs).

Symbols in package.json:
- ^1.2.3 → update minor + patch automatically.
- ~1.2.3 → update only patch automatically.

.gitignore:
- Used to ignore files/folders while pushing code to Git.
- Example: node_modules, .env, build files.

Parcel (Bundler)
Command:
- npx parcel index.html → starts dev server on localhost:1234.
- npm install -D parcel → install parcel locally.

What Parcel does (Key Features):
- Dev build
- Local server
- HMR (Hot Module Replacement)
- File watching algorithm (C++)
- Caching → faster builds
- Image optimization
- Minification
- Bundling
- Compression
- Consistent hashing
- Code splitting
- Differential bundling
- Diagnostics
- Error handling
- HTTPS support
- Tree shaking (remove unused code)

Output Folder:
- dist → contains production or dev build files generated by Parcel.

React Installation:
- npm install react
- npm install react-dom

Imports:
- import React from "react";
- import ReactDOM from "react-dom";
⚠️ If using inside <script> tag → add type="module".

Browserslist:
- "browserslist": { "last 2 versions" }
- Ensures app works on the latest 2 versions of all browsers.
